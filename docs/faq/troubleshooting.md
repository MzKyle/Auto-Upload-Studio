# 故障排查 FAQ

## 日期或工作次目录没有出现

- 启用的项目 Profile 中是否添加了日期目录的父级扫描目录。
- 自动扫描只处理当天日期目录；旧日期补传需手动添加具体工作次目录。
- 日期名是否为有效 `YYYY-MM-DD`。
- 工作次目录名是否匹配配置的 `workDirNamePattern`，默认 `HH-MM-SS`。
- 数据是否放在日期目录的直接子目录。
- 文件是否仍在写入，导致稳定性检查未通过。
- Profile 是否停用、目标云是否选错，或上传时间窗口是否尚未打开。

日期目录根部文件不会上传。工作次目录内部新增或修改文件会持续检测并增量上传。
`teach` 等不匹配工作次正则的目录会显示为“已忽略目录”，需要上传时点击“恢复监控”。

## 云端连接测试失败

检查 Endpoint、Region、Bucket、AK/SK、网络和列举权限。腾讯云还应检查 Endpoint
协议、path-style 服务兼容性和 TLS 证书；只有确认使用无法验证的自签名证书时才
开启“不安全 TLS”。

连接测试成功但上传失败通常是缺少对象写入或分片权限，也可能是本地文件被删除、
网络超时或服务端限流。

## 双云只有一端成功

逻辑任务会保持失败，成功端保持 `completed`。切换到失败云端标签页并执行“重试此
云端”，系统只会重置和上传该端未完成文件。

## 日期目录没有封账

必须同时满足：

- 日期早于当前本地日期。
- 至少发现一个焊接子目录。
- 每个逻辑任务完成。
- 双云任务的两个目标都完成。

失败或暂停任务会使日期状态为 `blocked`；已跳过任务不会阻塞，但日期会显示
“已完成（含跳过）”。

## 源目录已被删除

未上传完成的工作次目录被删除后，文件监控或下一次周期校准会取消上传，并标记
“已跳过（源目录已删除）”。已全部同步的任务保持成功状态。详情页可查看未上传文件；
只有源目录重新出现后才能执行“恢复监控”。

## SFTP 内存高

当前实现会将单个远程文件完整读入 Buffer。大文件或海量文件应改用 rsync，让普通
任务链路处理分片、恢复和历史。

## 自动清理会删除什么

清理服务优先删除超过保留天数的已封账日期目录，然后处理独立的已完成
`local` / `rsync` 任务。`manual`、失败、暂停、未完成和双云部分完成的数据不会
自动删除。

历史记录删除只删除数据库记录，不删除本地文件。

## Node.js 报语法错误

项目要求 Node.js 18+。Node.js 12 会在 TypeScript、`tsx` 或依赖代码中的现代语法
处失败。升级到 Node.js 20 LTS 后重新执行安装和验证。
