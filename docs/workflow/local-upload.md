# 本地目录上传

## 目录约定

在设置中添加日期目录的父级数据根目录：

```text
/data/upload-root/
  2026-06-18/
    04-39-04/
      camera1/0001.jpg
      metadata.json
```

扫描器只把 `04-39-04` 这样的直接子目录作为任务。单个焊接目录完成后不再检测
内部增量；后续采集必须创建新的焊接子目录。

## 操作步骤

1. 选择仅阿里、仅腾讯或双云上传。
2. 配置并测试所有启用的云端。
3. 添加 `/data/upload-root` 这类数据根目录。
4. 配置稳定性检查、并发和时间窗口。
5. 触发扫描或等待自动扫描。
6. 观察日期汇总和当前云端标签页中的任务进度。
7. 失败时在对应云端视图中执行“重试此云端”。

## 对象路径

每个云端使用自己的 Prefix：

```text
{providerPrefix}/2026-06-18/04-39-04/camera1/0001.jpg
{providerPrefix}/2026-06-18/04-39-04/metadata.json
```

任务创建后修改 Prefix 不会改变该任务的目标路径。

## 完成与封账

- 单云任务：选定云端完成后逻辑任务完成。
- 双云任务：阿里和腾讯都完成后逻辑任务完成。
- 日期目录：跨天且全部已发现焊接任务完成后写入 `day_upload.json`。
- 迟到目录：自动撤销封账，创建新任务，全部完成后重新封账。
- 自动清理：只有逻辑完成并满足保留天数后才会删除本地数据。

## 任务控制

| 操作 | 行为 |
| --- | --- |
| 暂停 | 中止运行中上传，未完成文件保留为可恢复状态 |
| 恢复 | 将任务重新排队 |
| 取消 | 将任务标记为失败 |
| 重试此云端 | 仅重置指定失败目标，成功云端不会重传 |
