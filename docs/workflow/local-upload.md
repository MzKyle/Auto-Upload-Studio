# 本地目录上传

## 适用场景

本地采集程序按“数据根目录 / 日期 / 单次焊接”写入数据，应用负责逐次上传并在跨天后汇总封账。

## 操作步骤

1. 在“设置”页配置 OSS，并测试连接成功。
2. 在“扫描配置”中添加数据根目录。
3. 设置扫描间隔和稳定性检查参数。
4. 根据需要配置上传时间窗口。
5. 回到“任务面板”，点击“触发扫描”或等待自动扫描。
6. 采集程序生成新的子目录。
7. 扫描器等待目录稳定后创建任务。
8. 队列在允许时间内启动上传。
9. 上传完成后在历史记录中查看结果。

## 目录示例

配置扫描目录：

```text
/home/kyle/sany/data
```

新增批次：

```text
/home/kyle/sany/data/2026-06-18/04-39-04/
  camera1/0001.jpg
  camera1/0002.jpg
  metadata.json
```

OSS prefix 为 `upload/` 时，对象路径：

```text
upload/2026-06-18/04-39-04/camera1/0001.jpg
upload/2026-06-18/04-39-04/camera1/0002.jpg
upload/2026-06-18/04-39-04/metadata.json
```

## 增量说明

每次焊接必须创建新的直接子目录。单个焊接目录稳定并上传完成后不再检测内部增量；日期目录可以持续出现新的焊接子目录。日期封账后出现迟到子目录时会自动重开并补传。

## 任务控制

| 操作 | 行为 |
| --- | --- |
| 暂停 | 取消当前任务，运行中文件回退 pending，任务变为 paused |
| 恢复 | 任务变回 pending，等待队列再次启动 |
| 取消 | 任务标记 failed，错误为“用户取消” |
| 重试 | failed 任务变回 pending，未完成文件重新上传 |

## 验证点

- 目录写入未完成时不会立刻生成任务
- 稳定检查通过后出现 `tmp_upload.json`
- 任务上传中出现 `process_task.json`
- OSS 中对象路径符合 prefix/folder/relativePath
- 暂停后可恢复，已完成文件不会重复计数
