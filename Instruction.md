# 数据采集上传工具验收说明

> 适用于应用版本 `2.0.2`，用于验证日期目录扫描、单云/双云上传、恢复和封账。

## 1. 环境

- Node.js 18+，建议 20 LTS
- npm 9+
- 至少一个可写的阿里云 OSS 或腾讯 TurboS3 Bucket
- 远程测试可选安装 `rsync`、`sshpass`

```bash
npm install
npm run dev
```

代码级检查：

```bash
npm test
npm run typecheck
```

## 2. 准备测试数据

使用早于测试当天的日期，便于验证封账：

```bash
mkdir -p /tmp/upload_test/2026-06-18/04-39-04
echo "hello" > /tmp/upload_test/2026-06-18/04-39-04/a.txt
echo '{"ok":true}' > /tmp/upload_test/2026-06-18/04-39-04/meta.json
```

设置中应添加 `/tmp/upload_test`，不能直接添加日期目录。

## 3. 配置云端

1. 选择仅阿里、仅腾讯或双云模式。
2. 填写每个启用云端的 Endpoint、Region、Bucket、Prefix 和 AK/SK。
3. 分别执行连接测试。
4. 腾讯云默认保持 TLS 校验；只有使用无法验证的自签名证书时开启不安全 TLS。

## 4. 扫描与上传

触发扫描后确认：

- 焊接目录经过稳定性检查再创建任务。
- `04-39-04` 中出现 `tmp_upload.json` 和 `process_task.json`。
- 对象路径为 `{providerPrefix}/2026-06-18/04-39-04/...`。
- 双云模式下，两个云端都完成后逻辑任务才完成。
- 任务面板和历史页可以切换阿里/腾讯标签。

## 5. 分云失败恢复

人为让一个云端配置失败并保留另一个云端可用：

- 成功端应保持完成。
- 逻辑任务应显示失败。
- 在失败云端标签页点击“重试此云端”。
- 修复配置后仅失败端补传，成功端不重传。

## 6. 日期封账与迟到目录

早于当前日期的测试目录全部完成后：

- 日期目录出现 `day_upload.json`。
- 日期汇总状态为完成。

随后新增：

```bash
mkdir -p /tmp/upload_test/2026-06-18/05-10-00
echo "late" > /tmp/upload_test/2026-06-18/05-10-00/late.txt
```

再次扫描后，旧封账应被撤销；新任务完成后重新生成 `day_upload.json`。

## 7. 其他验收

- 上传窗口只限制新任务，不中断运行中任务。
- 暂停后可恢复，取消后任务失败。
- rsync 拉取成功后创建普通任务。
- SFTP 和标注按当前模式返回逐云结果，但不创建普通任务历史。
- 自动清理只处理满足保留期的已封账日期或独立 `local` / `rsync` 任务。
- 手动添加任务不会被自动清理。
